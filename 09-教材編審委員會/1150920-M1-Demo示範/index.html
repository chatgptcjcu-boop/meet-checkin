第一次會議｜編審流程 Demo 示範腳本

115年9月20日（暫定）｜主辦帶領｜示範單元 M1-1.1｜約 85 分鐘

「各位委員好，今天不請大家立刻寫完整本教材。我會先帶大家走過一個單元從大綱到評量的完整步驟，走過一次後，下次會議請各位認領自己的單元，在線上一起寫。」

會前準備（主辦）

  • 投影：本頁、示範單元模板、協作平台
  • 預先建立 M1-1.1 協作文件（Demo 4 用）
  • 印發：委員閱讀須知（待產）、ABCD 一頁式 cheat sheet
  • 評核會第二次 ABCD 決議摘要（若已完成）
Demo 1｜15 min｜單元在大綱中的位置
「M1-1.1 是 86 單元中的第一課，屬於『組織治理』模組的緒論，目的是讓學員理解為何寺廟需要法律定位。」

操作：

  • 打開 08-評核委員會/1150630/units-86.json 或課程架構圖
  • 說明：模組 → 單元 → 時數 → 對應 T1 行為指標
  • 展示 單元模板 各區塊用途
Demo 2｜25 min｜ABCD 教學目標（現場共填）
「請大家看著模板，我們一起把四個欄位填起來。B 要用可觀察的動詞，例如『說明』『辨識』，不要只寫『了解』。」
A 對象宮廟管理師培訓學員（具基礎廟務經驗）
B 行為（現場填入）例：說明寺廟法人登記的三種型態差異
C 條件（現場填入）例：給定三種組織型態簡介資料
D 標準（現場填入）例：正確指出各型態 2 項以上關鍵差異
Demo 3｜20 min｜評量設計
「教學目標寫好後，要問：怎麼知道學員真的會了？形成性是學習中，總結性是單元結束。」
  • 形成性：課中個案討論、小組比對法人型態表
  • 總結性：3 題情境選擇＋1 題簡答（範例題幹現場起草）
  • 強調：評量必須對應 B 欄動詞
Demo 4｜15 min｜協作平台操作
「各位認領單元後，會在這個平台共寫。我示範怎麼開文件、邀請協作、留修訂建議。」

操作：

  1. 開啟 協作平台 實際連結
  2. 貼上 Demo 2–3 內容
  3. 示範「建議模式」留言一則
  4. 說明版本紀錄與命名規則：M1-1.1-緒論-主筆姓名
收尾｜10 min｜認領預告

發布 單元認領矩陣，說明第二次會議認領規則與交稿時程。

會後產出

  • M1-1.1 示範完稿（作為全場參考範例）
  • 委員理解 ABCD＋評量＋平台三要素
  • 第二次會議通知（單元認領）
