# 第一次評核委員會｜資料彙整工作流程

## 目的

將現場委員紙本《填答手冊》（正文＋附錄）掃描歸檔，逐人、逐章彙整意見，作為第二次評核委員會（115/08/30）的討論基礎與 icapV75 附錄十一素材。

## 資料夾結構

```
1150630-data-compilation/
├── index.html              ← 彙整總入口
├── members.json            ← 委員名單與狀態（請先更新姓名）
├── WORKFLOW.md             ← 本文件
├── 00-compilation-overview.html        ← 進度儀表板
├── 01～06 彙整頁            ← 跨委員主題彙整
└── members/
    └── M01-委員姓名待填/
        ├── index.html      ← 該委員專區
        ├── NOTES.md        ← 鍵入摘要／OCR 備註
        └── scans/          ← 掃描 PDF／影像放這裡
```

## 步驟一：更新委員名單

1. 開啟 `members.json`，將 `M01`～`M10` 的 `name`、`unit`、`attendance` 改為實際資料。
2. 將資料夾重新命名，例如 `M01-委員姓名待填` → `M01-王小明`（同步更新 `members.json` 的 `folder` 欄）。
3. 若委員超過 10 人：複製 `members/_TEMPLATE/` 為 `M11-姓名`，並在 `members.json` 新增一筆。

## 步驟二：上傳掃描檔

每位委員的 `scans/` 建議檔名：

| 檔名 | 內容 |
|------|------|
| `01-正文-封面至貳.pdf` | 正文冊 壹～貳 |
| `02-正文-叁至肆.pdf` | 正文冊 叁～肆 |
| `03-正文-伍至陸.pdf` | 正文冊 伍～陸 |
| `04-附錄-86單元.pdf` | 附錄冊 86 單元標記表 |

- 建議 300 dpi、PDF 或 PNG/JPG。
- 掃描完成後，在該委員 `index.html` 將狀態改為「已上傳」，並在 `members.json` 更新 `scanStatus`。

## 步驟三：請 Agent 協助彙整

掃描上傳後，在 Cursor 對話中提供：

```
請彙整 08-evaluation-committee/1150630-data-compilation/members/M03-王小明/scans/
對照 Phase 模板 1150630/，填入該委員各章答案摘要。
```

Agent 將：

1. 讀取掃描（若為 PDF／圖片可 OCR 或您口述重點）。
2. 在 `members/Mxx/NOTES.md` 寫入結構化摘要。
3. 更新 `01～06` 彙整頁的跨委員統計（共識、分歧、主審認領等）。
4. 標記需第二次會議討論的「保留意見」與「86 單元爭議」。

## 步驟四：跨委員主題彙整（會議前 7 日目標）

| 彙整頁 | 對應手冊章節 | 第二次會議用途 |
|--------|--------------|----------------|
| 01-出席與基本資料 | 壹 | 附錄一、出席確認 |
| 02-需求與問題共識 | 貳 | 決議一佐證、Top 3 問題 |
| 03-七大範疇與任務 | 叁～肆 | 決議二、修訂清單 |
| 04-86單元必要度 | 附錄 | 決議三、ABCD 優先序 |
| 05-決議與主審認領 | 陸 | 模組分工、第二次議程 |
| 06-保留意見與待議 | 全冊 | 主席引導討論清單 |

## 步驟五：銜接第二次會議

彙整定稿後：

1. 主席審閱 `00-compilation-overview.html` 與 `05-resolution-claims.html`。
2. 工作團隊依主審認領結果，在 `1150830/` 預填各模組討論素材。
3. 定稿納入 icapV75 附錄十一；第二次會議產出附錄十二。

## 與 meet-checkin 簽到系統的關係

- **remote-members**：第一次會議可能有線上填答（試算表「填答紀錄」分頁）；可匯出 JSON 併入對應 `members/Mxx/NOTES.md`。
- **現場委員**：以紙本掃描為主；本彙整區專為現場回收手冊設計。
