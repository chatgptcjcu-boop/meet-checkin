# 教材編審委員會｜Demo → 協作工作流程

## 背景

編審委員多數不熟悉「教材編寫步驟」與「評量方法」。因此**第一次會議不由委員自行摸索**，而由主辦（您）**現場 Demo 完整走過一遍**，再請委員認領單元、於線上協作平台共寫。

## 五階段流程

```
Demo 示範 (M1) → 單元認領 (M2) → 線上協作撰寫 → 中期審閱 (M3) → 定稿驗收 (M4)
```

## 階段一：Demo 示範（115/07/30）｜2 小時

**目標：** 帶領未受過 iCAP 職能導向課程的講師，**逐步完成示範單元 M1-1.1 全流程**（對齊評核委員會 Phase 0–5 系統性架構）。

| Phase | 內容 | 時間 | 文件 |
|-------|------|------|------|
| Phase 0 | 身分與單元定位 | 10 min | `1150730-m1-demo/01-Phase0-*.html` |
| Phase 1 | 職能對位・教學內容大綱 | 20 min | `02-Phase1-*.html` |
| Phase 2 | ABCD 教學目標 | 20 min | `03-Phase2-*.html` |
| Phase 3 | 教學活動・學習單產出 | 20 min | `04-Phase3-*.html` |
| Phase 4 | 評量（考古題・勾選・分數觀察） | 25 min | `05-Phase4-*.html` |
| Phase 5 | 協作平台建檔・認領預告 | 25 min | `06-Phase5-*.html` |

主辦腳本：`1150730-m1-demo/DEMO-SCRIPT-host.md`  
委員學習單：`1150730-m1-demo/worksheets/`

**示範單元：** `M1-1.1` 緒論－為何寺廟需要法律定位？（見 `textbook-units/m1-1-1-demo-unit/`）

## 階段二：單元認領（115/10/18）

1. 開啟 `unit-claim-matrix.html`（86 單元 × 主筆／協作／審閱）
2. 每位委員認領 1～3 單元（依專長與評核會主審認領對齊）
3. 建立協作平台資料夾／文件（每單元一文件）
4. 訂定交稿里程碑（建議：M3 前 80% 完稿）

## 階段三：線上協作撰寫（M2～M3 之間）

**建議平台（擇一，於 `collaboration-platform/index.html` 設定實際 URL）：**

| 平台 | 適合情境 | 占位狀態 |
|------|----------|----------|
| Google 文件（共用資料夾） | 委員已熟悉 Google、需留言修訂 | 預設推薦 |
| HackMD / Notion | 需版本紀錄、嵌入媒體 | 备选 |
| GitHub + Markdown | 技術團隊、版本控管嚴謹 | 可與 meet-checkin 分離 |

**每單元文件結構（模板）：**

1. 單元資訊（代碼、時數、對應職能）
2. 學習目標（ABCD）
3. 教學內容大綱
4. 教學活動與媒體
5. 形成性評量
6. 總結性評量
7. 參考資料
8. 修訂紀錄

## 階段四：中期審閱（115/11/15）

- 盤點 `單元認領矩陣` 完成率
- 同儕審閱：A 委員審 B 委員單元（交叉）
- 主審／編輯委員會統一格式意見

## 階段五：定稿驗收（115/12/20）

- 對照評核委員會職能基準與 86 單元必要度
- 評量工具一致性檢查
- 定稿簽核、移交出版或數位上架

## 與評核委員會的銜接

| 評核委員會產出 | 編審委員會使用 |
|----------------|----------------|
| 86 單元架構與必要度 | 認領優先序 |
| ABCD 教學目標（第二次會議） | Demo 與各單元撰寫依據 |
| assessment-design原則 | 各單元評量命題規範 |
| 模組主審認領 | 編審主審對應 |
