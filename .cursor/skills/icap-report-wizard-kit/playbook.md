# iCAP 報告書格式精靈 — 主辦／講師現場 SOP

繁體中文操作手冊。技術複製細節見 [SKILL.md](SKILL.md) 與 [reference.md](reference.md)。

---

## 適用對象

- **主辦（教育委員會／秘書）**：開啟投影、確認試算表回傳、會後歸檔
- **講師／教材委員**：跟簡報填學習單、提交或列印 PDF

本模組為 **備援訓練章節**，不限單一評核或編審會議日期，可獨立排課使用。

---

## 會前準備（主辦，約 30 分鐘）

### 1. 確認連結可用

| 用途 | 建議開啟頁面 |
|------|--------------|
| 會場投影 | `report-wizard-slideshow.html`（全螢幕 F11） |
| 講師填寫 | `worksheets/instructor-worksheet-online.html` |
| 主辦總覽 | `dashboard.html` →「報告書格式精靈」 |

請在會前用 **硬刷新**（Cmd+Shift+R）各開一次，確認非 404。

### 2. 確認 Google 試算表

1. 開啟綁定 GAS 的試算表
2. 確認有分頁 **`icap-report-worksheet`**
3. 若為全新環境：請技術人員在 Apps Script 執行 **`testReportWizardWorksheet`**，應出現一列測試資料（可刪）

### 3. 準備講師端

- 每位講師需可連網的筆電或平板（提交學習單用）
- 可選：印出分段學習單（`worksheets/section-01`～`05`）供手寫草稿
- 提醒講師手邊準備：職能基準草稿、課程規劃表、ADDIE 各階段筆記

## 現場流程（建議 90–120 分鐘）

### 階段一：導覽（約 40 分鐘）

1. 主辦投影 **格式精靈簡報**（18 頁）
2. 操作方式：
   - **← →** 或空白鍵切頁
   - 下拉選單可跳至特定章節
   - 數字鍵 **3** 可快速跳到「報告書總地圖」（若已設定 hotkey）
3. 重點停留頁：
   - 第 3 頁：報告書總地圖（ADDIE 對位）
   - 第 6–8 頁：壹、課程資訊
   - 第 9–14 頁：表1–12
   - 第 17 頁：**交件前 Checklist**（請全場一起過一遍）

### 階段二：實作填寫（約 40 分鐘）

1. 請講師開啟 **講師學習單｜線上填寫版**
2. 建議填寫順序：
   - 封面基本資料（姓名、單位、課程名稱）
   - 壹、課程資訊（需求／簡介／目的）
   - 表1–2（分析）
   - 表3–5（設計）
   - 表6–8（資源與評量）
   - 表9–12（實施評估）
3. 不必一次寫完；可先填關鍵欄位，其餘會後補
4. 若網路不穩：改用手寫分段學習單，會後再登打

### 階段三：提交與備份（約 10 分鐘）

1. 講師點頁底 **「提交學習單」**
2. 出現成功視窗即表示資料已送出（實際寫入以主辦試算表為準）
3. **建議備份（可選）**：
   - 點「列印／存 PDF」留存個人副本
   - 檔名建議：`icap-report_報告書學習單_姓名_日期.pdf`
4. 主辦即時刷新試算表，確認有新列

### 階段四：收尾（約 10 分鐘）

1. 再次提醒：**iCAP 平台線上申請** 欄位可能與 ODT 報告書不同，交件前須至平台核對
2. 說明會後主辦將匯出試算表供委員會追蹤
3. 收集問題，排定下一場深度諮詢（若需要）

## 會後歸檔（主辦）

```
活動資料夾/
├── 01_報告書學習單試算表/     ← icap-report-worksheet 分頁匯出 CSV
├── 02_講師PDF備份/            ← 講師 Line 或 Email 回傳（若有）
└── 03_簡報投影紀錄/           ← 可選：會議記錄、簡報第 17 頁自查表截圖
```

試算表 **「結構化答案(JSON)」** 欄為完整填答備份，無需講師另交 JSON。

## 常見問題（現場）

| 問題 | 回應 |
|------|------|
| 按提交沒反應 | 檢查 Wi‑Fi；仍可用列印 PDF 交回 |
| 成功視窗有、試算表沒有 | 通知主辦；技術人員查 GAS URL 與部署 |
| 簡報變成整頁捲動 | CSS 未載入，改開啟正確 GitHub Pages URL 或換瀏覽器 |
| 欄位與我手邊 ODT 不一樣 | 本精靈依標準表1–12；特殊欄位請用 Word ODT 補充 |
| 和 iCAP 平台差在哪 | 內容應一致；平台欄位名稱／上傳步驟不同，交件前雙邊核對 |

## 推廣／複製到其他職能導向計畫

交付給新計畫主辦時，請一併提供：

1. 本 playbook（繁中 SOP）
2. `ASSET-PACK.md`（資產清單與 URL）
3. 已部署的試算表（複本）與 GAS 部署說明
4. 客製化後的 GitHub Pages 連結（簡報 + 線上學習單）

**保留 generic、僅改文案**：簡報結構、學習單欄位、GAS schema。  
**必須重綁**：試算表、GAS URL、主辦單位名稱、課程預設值、對外連結。
